# Mods 与集成

LeviLauncher 支持超出“基础启动器”范围的 Mod 相关工作流。

## 当前可用能力

- 导入 `.zip` Mods
- 导入 `.dll` Mods
- 启用或禁用已安装 Mods
- 删除不再需要的 Mods
- 使用 CurseForge 相关浏览与获取流程
- 使用 LIP 包管理流程
- 配合 LeviLamina 等加载器工作流

::: warning 这是高级区域
Mods、加载器与包管理集成的兼容性变化往往更快，尤其是在 Minecraft 预览版（Preview）上。请把这里当作高级能力，逐步测试。
:::

## 推荐的 Mod 工作流

1. 从一个干净的隔离版本开始。
2. 先确认该版本在无 Mods 情况下可以正常启动。
3. 一次只增加一个 Mod 或一小组改动。
4. 每次改动后都重新测试启动。
5. 重要世界始终保留备份路径。

## CurseForge

启动器提供 CurseForge 相关浏览与包流程，帮助你在应用内更方便地寻找合适资源。

适合以下需求：

- 需要更引导式的查找体验
- 想更快看到某个项目的兼容文件
- 不想所有搜索都在启动器外完成

## LIP 与 LeviLamina

这类集成更适合希望使用包驱动或加载器驱动工作流的高级用户。

最佳实践：

- 先从稳定的正式版（Release）版本开始
- 不要一次变更多个高风险因素
- 记录下“最后一次成功工作”的版本组合

## 如果加了 Mods 后出错

- 先禁用最新加入的 Mod
- 与上一个可正常工作的状态进行对比
- 进一步测试前先转移或备份重要世界
- 然后继续查看[更新与故障排查](./update-troubleshooting)
