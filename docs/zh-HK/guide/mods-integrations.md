# Mods 與集成

LeviLauncher 支援超出「基礎啟動器」範圍的 Mod 相關工作流程。

## 當前可用功能

- 匯入 `.zip` Mods
- 匯入 `.dll` Mods
- 啟用或停用已安裝的 Mods
- 刪除不再需要的 Mods
- 使用 CurseForge 相關瀏覽與獲取流程
- 使用 LIP 包管理流程
- 配合 LeviLamina 等加載器的工作流程

::: warning 這是進階區域
Mods、加載器與包管理集成的相容性變化往往較快，尤其是在 Minecraft 預覽版（Preview）上。請將此處視為進階功能，並逐步進行測試。
:::

## 建議的 Mod 工作流程

1. 從一個乾淨的隔離版本開始。
2. 先確認該版本在無 Mods 情況下可以正常啟動。
3. 一次只增加一個 Mod 或一小組改動。
4. 每次改動後都重新測試啟動。
5. 重要世界始終保留備份路徑。

## CurseForge

啟動器提供 CurseForge 相關瀏覽與包流程，協助你在應用程式內更輕鬆地尋找合適資源。

適合以下需求：

- 需要更具引導式的搜尋體驗
- 想更快查看某個項目的相容檔案
- 不希望所有搜尋都在啟動器外完成

## LIP 與 LeviLamina

這類集成更適合希望使用包驅動或加載器驅動工作流程的進階用戶。

最佳實踐：

- 先從穩定的正式版（Release）版本開始
- 不要一次變更多個高風險因素
- 記錄下「最後一次成功運作」的版本組合

## 如果加了 Mods 後出錯

- 先停用最新加入的 Mod
- 與上一個可正常運作的狀態進行對比
- 進一步測試前先轉移或備份重要世界
- 然後繼續閱覽[更新與故障排查](./update-troubleshooting)
