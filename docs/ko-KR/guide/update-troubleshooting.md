# 업데이트 및 문제 해결

LeviLauncher의 설치, 업데이트 또는 실행 동작이 예상과 다를 경우, 이 페이지를 우선 참조하세요.

## 일반적인 문제

### Gaming Services 누락 또는 손상

- 런처에서 제공하는 안내를 사용하여 설치하거나 복구합니다.
- 필요할 경우 Microsoft Store에서 Minecraft Bedrock의 설치 상태를 확인합니다.

### GameInput 누락

- LeviLauncher 안내에 따라 필요한 redistributable을 설치합니다.
- 설치 완료 후, 필요에 따라 런처를 다시 시작합니다.

### 설치 경로에 쓰기 불가

- **Settings**에서 호스팅 콘텐츠 경로를 쓰기 가능한 위치로 변경합니다.
- 런처가 설치 또는 자체 업데이트 시 관리자 권한을 요청하면, 신뢰할 수 있을 때만 실행을 허용합니다.

### 특정 버전이 실행되지 않음

- 먼저 해당 버전이 모드 없이 실행되는지 테스트합니다.
- 필요한 Windows 구성 요소가 여전히 존재하는지 확인합니다.
- 해당 버전이 완전히 설치되었는지 확인합니다.
- 깨끗한 격리 테스트 환경에서 문제를 재현해 봅니다.

### 다운로드가 매우 느리거나 반복적으로 실패

- 미러를 전환합니다.
- 잠시 후 다시 시도합니다.
- 조건이 허락된다면 로컬 패키지 소스로 변경합니다.

## 더 안전한 복구 단계

1. 중요한 월드를 먼저 백업합니다.
2. 최근에 추가한 모드 또는 리소스 팩을 제거합니다.
3. 깨끗한 격리 버전으로 대조 테스트를 수행합니다.
4. 필요할 경우 영향을 받은 버전을 다시 설치합니다.

## 자체 업데이트에 대하여

LeviLauncher는 애플리케이션 업데이트를 확인, 다운로드 및 설치할 수 있습니다. 설치 디렉터리에 쓰기 권한이 없을 경우, 일부 환경에서는 관리자 권한이 필요할 수 있습니다.

## 언제 Issue를 제출해야 하나요?

다음 조건을 모두 충족할 때 GitHub에 문제를 제출하는 것이 좋습니다:

- 문제를 안정적으로 재현할 수 있을 때
- 기본 복구 단계를 수행한 후에도 여전히 문제가 존재할 때
- 문제가 일반적인 Windows 환경 문제보다 LeviLauncher 자체에 더 가까워 보일 때

제출 시 다음을 가능한 한 첨부해 주세요:

- Windows 버전
- LeviLauncher 버전
- 구체적인 작업 단계
- 스크린샷 또는 로그(있는 경우)

문제 피드백 진입점: [GitHub Issues](https://github.com/LiteLDev/LeviLauncher/issues)
